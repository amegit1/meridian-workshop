R1 — Reports remediation

Fixing what was left
unfinished

The RFP references at least eight logged defects. We treat that as a floor, not a ceiling. Our audit will cover:

  • All four filter dimensions (Time Period, Warehouse, Category, Order Status)
  • Internationalization string gaps
  • Inconsistent API response shapes
  • Console errors and pattern inconsistencies

What "fixed" means

Not "works on my machine." Fixed means: passing automated tests, consistent behavior across warehouses, and no console errors. We won't close a defect without a test that would have caught it.

Budget ceiling

Current stock · demand forecast · suggested quantity · unit cost · total cost. Everything needed to approve the order.

Built using the same Vue 3 Composition API patterns as the rest of the application. Same filter bar. Same warehouse selector. No new frameworks.

R3 — Automated testing

The thing that unblocks
everything else

We use Playwright — the same browser automation tool trusted by Microsoft, Google, and most major software teams.

Coverage includes:

  • Dashboard — key metrics load correctly
  • Inventory — filtering by warehouse and category
  • Orders — filtering by status and time period
  • Reports — all previously broken filter combinations
  • Restocking — budget input and recommendation output

Committed to the repo

Tests live in your repository. Your IT team can run them at any time with a single command. We include a README explaining how.

What this unlocks

Once IT has a passing test suite, they have a safety net. Future changes — by us or any future vendor — can be reviewed against a concrete definition of "working."
