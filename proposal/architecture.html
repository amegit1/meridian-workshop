Filter Data Flow

The global filter bar is a singleton — state is shared across all views via useFilters.js.

User changes filter (FilterBar.vue)
→
useFilters ref updated (selectedPeriod / selectedLocation / selectedCategory / selectedStatus)
→
View watches filter change
→
getCurrentFilters() builds params object
→
api.js appends query params to URL
→
FastAPI filters in-memory data
→
Vue re-renders with filtered data

Data Files

server/data/

  • inventory.json — SKUs, stock levels, reorder points, unit costs
  • orders.json — customer orders with status and line items
  • demand_forecasts.json — per-SKU demand trend data
  • backlog_items.json — unfulfilled order lines
  • purchase_orders.json — supplier POs linked to backlog items
  • spending.json — financial summary and transaction data

Data characteristics

  • All data loaded into memory at server startup via mock_data.py
  • Data mutations (POST) are in-memory only — server restart resets all state
  • No relational constraints enforced at the data layer
  • Date format: YYYY-MM-DD throughout
  • Warehouses: San Francisco, London, Tokyo

Tech Stack Summary

Frontend

  • Vue 3 Composition API (setup function)
  • Vite Dev server + build tool
  • Axios HTTP client (via api.js)
  • Vue Router Client-side routing
  • Custom SVG charts — no chart library dependency
  • CSS: scoped component styles + global in App.vue

Backend

  • Python 3.11+ Runtime
  • FastAPI Web framework
  • Pydantic Request/response validation
  • Uvicorn ASGI server
  • uv Package manager
  • CORS: allow-all (development only)

Known Limitations

🔴
No persistent database. All data lives in JSON files loaded at startup. Any write operations (create purchase order, add task) are lost on server restart. A production system would require a proper database (PostgreSQL recommended).
🔴
No authentication or authorization. The API accepts requests from any origin with no credentials. Suitable for internal development only — not production-ready.
🟡
CORS allows all origins. allow_origins=["*"] is set in the FastAPI middleware. Must be restricted before any production deployment.
🟡
Mixed API patterns in frontend. Most views use the centralized api.js client, but some components made direct axios calls. This inconsistency has been resolved in the R1 remediation.
🟡
Mixed component API styles. The majority of views use Vue 3 Composition API. The Reports view used Options API (the older pattern) — resolved in R1 remediation.
🟢
i18n infrastructure is in place. English and Japanese locale files exist. The useI18n composable is wired into most views. Extending to full coverage is straightforward (D2 scope item).
